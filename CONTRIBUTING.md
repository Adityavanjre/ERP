# Contributing to Nexus ERP

To maintain system integrity and stability, we follow a strict **Independent Branch Workflow**. Direct pushes to the `main` branch are prohibited and will be rejected by repository guards.

## 🛠️ How to Contribute

1.  **Fork the Repository**: Create a fork of the project to your own account.
2.  **Create a Feature Branch**: Always branch off from `main`. Use descriptive names:
    - `feature/new-module-name`
    - `fix/bug-description`
    - `docs/update-manual`
3.  **Implement Changes**: Ensure your code follows the existing patterns and passes all local linting.
4.  **Submit a Pull Request (PR)**:
    - Target the `main` branch of the original repository.
    - Provide a clear description of what changed and why.
    - Wait for code review and automated checks to pass.

## 🛡️ Protection Protocol
- **No Direct Hotfixes**: Even minor changes must go through the PR process.
- **Signed Commits**: Contributors are encouraged to sign their commits for verification.
- **Audit Logs**: All merges are recorded in the system audit logs for compliance.
